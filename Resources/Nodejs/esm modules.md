

## المرحلة الأولى: الـ Identity Crisis (إزاي نود بيعرف إنه ESM؟)

أول سؤال في الإنترفيو: **"يا خالد، نود سيرفر سايد وولد في بيئة CommonJS، إزاي فجأة بيقدر يشغل ESM من غير ما يضرب؟"**

نود مبيخمنش. فيه نظام صارم جداً عشان يحدد هو هيتعامل مع ملفك بـ "الرابر القديم" (CJS) ولا بـ "نظام الموديول الحديث" (ESM). العملية دي بتتم عن طريق 3 طرق بالترتيب:

### 1. الـ Extension (الكلمة الأولى والأخيرة)

- **`.mjs`:** لو سميت ملفك كدة، نود هيفهم أوتوماتيك إنه **ES Module** مهما عملت.
    
- **`.cjs`:** لو سميته كدة، نود هيفهم إنه **CommonJS** مهما عملت.
    
- **`.js`:** وهنا بقى "المربط" والمشاكل.. الـ `.js` ملوش هوية، نود بيروح يدور على هويته في الـ `package.json`.
    

### 2. خاصية `"type": "module"` (المنقذ)

لو رحت للـ `package.json` وكتبت السطر ده، نود هيعتبر أي ملف `.js` جوه الفولدر ده (والفولدرات اللي تحته) عبارة عن **ESM**.

> **تنبيه سينيور:** لو عملت كدة، كل الأكواد القديمة اللي فيها `require` هتضرب! لازم تقرر يا إما الأبلكيشن كله ESM يا إما CJS.

### 3. الـ `--input-type` Flag

دي بنستخدمها لو بنبعت كود لنود عن طريق الـ `stdin` (الترمينال) أو كود String، بنقوله `--input-type=module`.

## المرحلة الثانية: الـ Parsing Phase (لما نود يقرأ الكود قبل ما يشغله)

دي أهم حتة تفهمها عشان تبقى سينيور. الـ CommonJS كان "Runtime"، يعني نود بيقرأ السطر وينفذه، لو لقى `require` يروح يجيب الملف ويرجع يكمل.

**الـ ESM مختلف تماماً.. بيمشي في 3 مراحل منفصلة:**

1. **Construction (الـ Fetching):** نود بيقرأ الملف بتاعك، وأول ما يشوف كلمة `import` مبيستناش لما يوصلها في التنفيذ، هو بيجمع كل الـ imports اللي في الملف ويطلع "شجرة" (Module Map).
    
2. **Instantiation (الـ Linking):** نود بيجهز أماكن في الميموري لكل الـ variables اللي أنت عامل لها `export` و `import` ويوصلهم ببعض (بنشبهها بإنك بتركب "مواسير" الميموري قبل ما تضخ فيها الداتا).
    
3. **Evaluation (الـ Execution):** هنا بقى يبدأ يشغل الكود سطر بسطر.
    

**ليه ده عمق؟**

لأن ده معناه إن الـ `import` لازم تكون في الـ **Top-level**. مينفعش تحط `import` جوه `if` condition أو جوه `function` (زي ما كنا بنعمل في `require`). ليه؟ لأن نود محتاج يعرف الـ "شجرة" دي في مرحلة الـ Construction قبل ما البرنامج يبدأ أصلاً.

## المرحلة الثالثة: الـ Live Bindings (السحر اللي بيبهر الكل)

ده سؤال انترفيو ليفل "تنين". **"إيه الفرق بين الداتا اللي راجعة من `require` والداتا اللي راجعة من `import`؟"**

- **في الـ CommonJS:** الـ `require` بترجع **Copy** (نسخة). لو الموديول غير قيمة متغير جواه بعد ما أنت عملت له require، النسخة اللي عندك مش هتتغير.
    
- **في الـ ESM:** الـ `import` بترجع **Live Binding**. هي مش نسخة، هي "شباك" (Window) بيبص على المتغير الأصلي في الموديول التاني. لو الموديول التاني غير القيمة، القيمة هتتغير عندك في نفس اللحظة!

## المرحلة الرابعة: الـ Mandatory Extensions (نود مابقاش "بيألف")

في الـ CommonJS كنت بتكتب `require('./user')` ونود يدور على `user.js` أو `user.json` أو `user/index.js`.

**في الـ ESM، اللعب ده ممنوع.** نود اتبع مواصفات البراوزر (Browsers Standard).

- لازم تكتب الـ Extension كامل: `import { user } from './user.js';`
    
- لو ملقاش الملف بالظبط، هيطلع `ERR_MODULE_NOT_FOUND`.
    
- **ليه؟** عشان نود عايز الكود بتاعك يكون "Standard" يشتغل في الكروم وفي النود بنفس الشكل من غير ما يحتاج "Logic" معقد في البحث.

## المرحلة الرابعة: الـ `import.meta` (وريث العرش)

أول صدمة بتواجه أي حد بينقل من CJS لـ ESM هي لما يكتب `__dirname` أو `__filename` ويلاقي نود بيحدفه بـ `ReferenceError`.

**ليه نود شالهم؟**

لأن الـ ESM مصمم إنه يشتغل في "أي حتة" (نود، متصفح، دينو). الـ `__dirname` دي حاجة خاصة جداً بنظام الملفات بتاع نود. المواصفات العالمية للجافا سكريبت قالت: "إحنا هنعمل أوبجكت اسمه `import.meta` شايل معلومات عن الموديول الحالي، وكل بيئة (Environment) تحط فيه اللي هي عايزاه".

### إزاي نرجع الـ `__dirname` يا خالد؟

السينيور مش بيفضل يعيط على اللي فات، هو بيعرف يطلعه من الميموري. الـ `import.meta` جواها خاصية اسمها `url` بتبدأ بـ `file://`. وعشان نحولها لمسار نود بيفهمه، بنعمل كدة:

JavaScript

```
import { fileURLToPath } from 'url';
import { dirname } from 'path';

// ده البديل الرسمي لـ __filename
const __filename = fileURLToPath(import.meta.url);

// ده البديل الرسمي لـ __dirname
const __dirname = dirname(__filename);

console.log('أدي يا سيدي المسار اللي كنت بتدور عليه:', __dirname);
```

**عمق السينيور:** ليه اللفة دي؟ عشان الـ ESM ممكن يتحمل من "لينك" (URL) مش بس من الهارد ديسك (زي ما بيحصل في البراوزر)، فـ نود لازم يتعامل مع المسارات كـ URLs الأول.

## المرحلة الخامسة: الـ Top-level `await` (التحرر من العبودية)

دي بقى الحتة اللي بتخلي مطورين الـ CommonJS يغيروا مننا. زمان في CJS، لو عايز تنادي API أو تفتح DB في بداية الملف، كنت لازم تعمل كدة:

JavaScript

```
// النظام القديم CJS
async function start() {
    const data = await fetchDB();
}
start();
```

أو تستخدم `.then()` اللي بتخلي الكود شكله "سباجيتي".

**في الـ ESM، نود سمح لك تكتب `await` في "الهواء" (Top-level).**

نود بيوقف تنفيذ الموديول ده (وأي موديول بيعمله `import`) لحد ما الـ `await` تخلص.

**خطورة السينيور:** خلي بالك يا خالد، الـ Top-level await دي "سلاح ذو حدين". لو أنت عملت `await` لحاجة بتاخد وقت طويل قوي (زي داتا بيز مهنجة)، أنت بتوقف الـ **Bootstrapping** بتاع الأبلكيشن كله، لأن نود مش هيكمل تحميل باقي الموديولز غير لما الموديول ده "يخلص".

## المرحلة السادسة: الـ Interoperability (خناقة الإخوة الأعداء)

ده سؤال الإنترفيو اللي بيفصل العيل عن الراجل: **"إزاي نستخدم CommonJS جوه ESM والعكس؟"**

### 1. ESM ينادي CJS (سهلة بس غدارة)

ممكن تعمل `import` لملف `.cjs` عادي:

JavaScript

```
import logger from './my-old-logger.cjs';
```

**المشكلة:** نود بيتعامل مع الـ CJS موديول كأنه "Default Export" واحد. يعني لو ملف الـ CJS مطلع 10 حاجات، الـ ESM هيشوفهم كلهم كـ Object واحد، ومينفعش تعمل Named Import (يعني `import { x }` مش هتشتغل إلا لو نود قدر يحللها بطريقة خاصة).

### 2. CJS ينادي ESM (المهمة المستحيلة)

لو حاولت تعمل `require('./module.mjs')`.. نود هيضرب في وشك Error صريح: `ERR_REQUIRE_ESM`.

**ليه؟** لأن `require` عملية **Synchronous** (بتحصل في وقتها)، والـ ESM بطبعه **Asynchronous** (بيتحمل في مراحل). مينفعش حاجة "لحظية" تستنى حاجة "مستقبلية" وهي أصلاً مش مصممة لكدة.

**الحل الصايع:** عشان تنادي ESM من جوا ملف CJS، لازم تستخدم الـ **Dynamic Import**:

JavaScript

```
// جوه ملف .cjs
async function loadESM() {
    const myModule = await import('./module.mjs'); // دي شغالة زي الفل
}
```

## المرحلة السابعة: الـ Cache في الـ ESM

فاكر لما قلنا إن الـ `require.cache` في الـ CJS ممكن نمسح منه حاجات؟

في الـ ESM، **مفيش `require.cache`**. الـ Loader بتاع الـ ESM في نود بيستخدم "Internal Module Map" مش متاح ليك كمطور إنك تلعب فيه بسهولة.

**ده معناه إيه؟** معناه إنك لو عملت `import` لملف، هو هيفضل في الميموري طول عمر الأبلكيشن، ومقدرش "أمسحه" عشان أعمله Hot Reload بسهولة زي زمان. السينيورز هنا بيستخدموا تريكات زي إضافة `query string` للمسار (مثلاً `import('./file.js?update=' + Date.now())`) عشان يضحكوا على نود ويخلوه يحمله كأنه ملف جديد.

## تطبيق عملي (The Master Task)

تخيل يا خالد إنك بتعمل **Global Configuration Module** بالـ ESM، ومحتاج تـ Fetch الداتا من AWS Secrets Manager قبل ما أي حد في الأبلكيشن يبدأ يشتغل.

JavaScript

```
// config.mjs
console.log("1. جاري تحميل الإعدادات...");

// Top-level await! الأبلكيشن كله هيقف هنا لحد ما دي تخلص
const response = await fetch('https://api.secrets.com/db-config');
export const dbConfig = await response.json();

console.log("2. الإعدادات جاهزة!");
```

لو حد عمل `import { dbConfig } from './config.mjs'` في ملف `app.js`.. السيرفر مش هيطبع "App started" غير لما "الإعدادات جاهزة" تظهر الأول.

### أسئلة الإنترفيو لليفل "وحش الـ ESM":

1. **س: ليه الـ `require` ممنوعة في الـ ESM؟**
    
    - **ج:** لأن `require` بتشتغل Synchronous والـ ESM مصمم يكون Asynchronous عشان يواكب الـ Browser standards والـ Top-level await.
        
2. **س: إيه الفرق بين `import x from 'y'` و `import('y')`؟**
    
    - **ج:** الأولى Static Import (بتحصل قبل ما الكود يبدأ تنفيذ)، والتانية Dynamic Import (بتحصل في الـ Runtime وبترجع Promise).
        
3. **س: لو عندي ملف `.js` وعايزه يشتغل ESM من غير ما أغير الـ `package.json` بالكامل، أعمل إيه؟**
    
    - **ج:** أغير الـ extension بتاعه لـ `.mjs`.
